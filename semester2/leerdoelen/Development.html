Versiebeheer met Git

In het begin van het semester heb ik veel gehamerd op Git versiebeheer in mijn portfolio. Tijdens het maken van mijn portfolio heb ik bij bij elk ding wat ik voltooid onderdeel een Push gedaan. Ook voor iedere POC heb ik Push gedaan naar mijn Git repository.

Wat heb ik geleerd?

Ik heb snel geleerd dat versiebeheer heel erg handig is zodat je wanneer er iets mis gaat in je project of je hebt een fout gemaakt, je niet alles kwijt bent maar gewoon terug kan naar de vorige commit.

Wat kon beter?

Omdat ik net begon met git moest ik dit nog onder de knie krijgen en heb ik vaak dingen gepusht naar de verkeerde repository. Daardoor heb ik vaak opnieuw een repo moeten maken en verwijderen voordat alles er goed in stond. Ook heb ik geleerd dat de folder naam veranderen een heel gedoe is.

Wat zou ik de volgende keer anders doen?

Ik zou netter en zorgvuldiger te werk gaan in mijn git. Hierdoor kan ik kleine foutjes en verkeerde pushes zoveel mogelijk elimineren.

Business Cards

Voor de 2e opdracht die ik van de docent heb gekregen moest ik business cards maken. Later is deze opdracht uitgebreid met het kunnen uitlezen van een json file. Daarbij moest moest voor het groepsproject gegevens uit javascript kunnen pakken met json en daar kaartjes van genereren.

Daarbij ging het maken van het prototype heel soepel en heb ik daar 2 tutorials voor kunnen volgen. Bij het uitlezen van de json heeft de docent een kleine workshop gegeven over hoe je dit daadwerkelijk kan doen. Dit toegepast en met meerdere malen op je juiste manier googlen is mijn dit gelukt. Omdat ik hier redelijk snel mee klaar was wilde ik niet moeilijk maken voor mezelf. Ik wilde een clipboard maken waarmee je het e-mailadres per kaartje kon kopiëren. Omdat je hier de ID normaal aan toevoegt was het best moeilijk omdat ieder kaartje automatisch gegenereerd was. Ik heb helaas geen tijd meer gehad om de json hiervoor aan te passen. Wel is de code van de clipboard toegepast in de code van de HTML en CSS.

Business Cards POC

Wat heb ik geleerd?

Tijdens het maken van de business card heb ik het effect hover geleerd wat ik in mijn portfolio ook best vaak terug laat komen. Ook heb ik duidelijk geleerd hoe ik de kaartjes kan afronden zodat deze niet vierkant zijn. Ook ben ik gegroeid in het gebruik van CSS en HTML. De json opdracht was een leuke toevoeging en was ook lekker puzzelen om de json file de Javascript uit te laten lezen. Ook was het een uitdagingen om ervoor te zorgen dat de kaartjes werden aangemaakt. Vooral vind ik van mezelf dat het clipboard veel toegevoegde waarde had gehad als het had gewerkt. Ondanks dat het niet werkt weet ik wel hoe ik de clipboard moet toepassen in volgende projecten en POC’s.

Wat kon beter?

Doordat de Javascript de kaartjes genereerde staan er hier en daar minor bugs in omdat de HTML leeg stond. Daarbij heb ik in de CSS gezegd dat de content in het midden moest staan. En tijdens het genereren van de kaartjes dacht ik dat de Javascript code niet werkte. Na goed nadenken viel het kwartje dat ik in de CSS de marges in het midden heb gezet en dat alles kaartjes bovenop elkaar werden weergegeven waardoor het leek alsof er maar 1 kaartje werd gegenereerd. Nadat ik de flex container heb gebruikt was dit probleem snel opgelost. Hierdoor heb ik veel tijd verloren.

Wat zou ik de volgende keer anders doen?

Meer nadenken over de positionering in CSS om dingen zoals hierboven beschreven te hebben voorkomen.
