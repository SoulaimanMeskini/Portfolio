portfolio


Dit semester wilde ik me compleet focussen op een professionele portfolio website. Dit wilde ik ook in mijn eigen stijl neerzetten. Ook al is de doelgroep voor dit portfolio de docenten, Ik wilde niet iets maken wat ik na semester 2 weer weg leg. Dit portfolio maak ik zo zodat ik er elk semester op verder kan bouwen en waar al mijn vooruitgang in te zien zal zijn. Met dit in mijn gedachten ben ik bezig geweest met een mock-up view.


Feedback

Na met meerdere docenten gesproken te hebben over mijn portfolio zijn er een aantal conclusies naar voren gekomen. Namelijk dat het er donker uitzag maar dit wel goed bij de stijl past. Ik kreeg als tip mee om met tijd te spelen in de vorm van een dag en nachtmodus. En het logo moest meer naar voren springen. Ik kreeg ook als tip dat ik ook al is het portfolio bedoeld voor een lange periode, dat ik ook goed moet nadenken over bepaalde layouts en de leeruitkomsten van semester 2.


Docenten


Wat heb ik geleerd


Ik heb een sterke basis aangelegd in HTML en CSS. Veel groter dan in S1. Daarnaast heb ik vrijwel geen uitleg nodig gehad van leerlingen of ben ik tegen hele moeilijk problemen aangelopen. Wel heb ik een docent gevraagd om mij wekelijkse opdrachten te geven zodat ik wel op schema bleef en ook mijn tijd aan de goede dingen deed besteden. Dit heeft mij veel geholpen om een begin te maken aan Javascript en daar de basis van te begrijpen.


Wat kon er beter


Ik had een duidelijker stappenplan op moeten zetten of een beter schema moeten maken waarin ik opschreef wat ik aan het einde van de week af wilde/moest hebben zodat ik op het einde niet in de soep zou lopen met mijn portfolio en zodat ik nog extra feedback kon vragen aan docenten.


Wat zou ik de volgende keer anders doen


Ik zou de opdracht gerichter aangaan in plaats van in elke opdracht heel veel van mezelf te stoppen zodat ik ook alle criteria van de opdracht behaal.



Ruby Mus Website


Tijdens het brainstormen voor een website voor Ruby Mus is ons groepje bezig geweest met verschillende Wireframes. Na verschillende soorten samen te hebben behandeld hebben we aspecten bij elkaar gevoegd om zo onze eigen stijl te vinden. Nu we een duidelijk beeld hadden over hoe de website eruit zou moeten komen te zien, hebben we de one pager opgesplitst in 4 delen:
- Hoofdpagina
- Music Player
- Contact Page
- Events/Projects
We hebben deze taken verdeeld en ik heb de music player op de website gemaakt.


Wat heb ik geleerd


Doordat ik de music player moest maken heb ik veel geleerd over de styling met CSS en het gebruik van Javascript. Tijdens het maken van deze opdracht heb ik een eigen file opgezet. Daarbij kwam ik snel achter dat ik margins en benamingen in CSS verkeerd heb gebruikt. Ik heb op de body tag de margins en padding veranderd dit ging dus fout toen ik mijn project ging samenvoegen met de one pager. Hierdoor heb ik dus geleerd om andere benamingen gebruiken dan de standaard naming conventions (body, img., h3, p, etc.) Nadat ik de music player had gemaakt vond ik hem er nog een beetje saai en heb ik een bewegende gradient toegepast met CSS en Keyframes.


Wat kon er beter


Omdat ieder in zijn eigen file aan het werken was ipv in 1 hoofdbestand, ontstond er op een gegeven moment een probleem dat iedereen andere naamgeving had. Dit was voor de deadline onze valkuil omdat wij deze problemen snel moesten oplossen. Dit is iets waar wij daarna veel rekeningen mee hebben gehouden zodat we dit konden voorkomen of tijdig konden oplossen.


Wat zou ik de volgende keer anders doen


Een gezamenlijk git opstellen waardoor iedereen in 1 project kan samenwerken en beter nadenken over mijn benamingen in HTML en CSS. De volgende keer wil ik ook veel meer usability tests uitvoeren. Zo kan ik sneller eventueel bugs eruit halen en controleren of mijn project wel goed aansluit op de doelgroep.
